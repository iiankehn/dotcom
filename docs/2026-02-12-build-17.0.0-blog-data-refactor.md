## Build 17.0.0 – Blog data refactor

- **Date**: 2026-02-12
- **Area**: Blog (`/blog` and `/blog/archive`)

### Summary

- **Introduced a shared web “database” for posts** using a new file `blog/posts.js`.
- **Refactored `blog/index.html`** to render its list of posts dynamically from that shared data.
- **Refactored all archive post pages** under `blog/archive/` to read their content from the same data source.

### Details

- **New file**: `blog/posts.js`
  - Defines `window.BLOG_POSTS`, an array of post objects.
  - Each post includes:
    - `slug`: unique identifier (e.g. `chatgpt-may-be-coming-to-carplay`)
    - `title`
    - `publishedOn` (ISO date string) and `publishedLabel` (display string)
    - `hasArchive` and `archivePath` for posts that have full article pages
    - `indexHtml`: HTML snippet used on the main blog index
    - `articleHtml`: full article HTML used on the archive page
  - This replaces hardcoded post content in individual HTML files and acts as the single source of truth for blog copy.

- **Updated**: `blog/index.html`
  - Replaced the hardcoded list of posts in the `<header>` with an empty container: `<header id="blog-posts">`.
  - Loads `posts.js` via `<script src="posts.js"></script>`.
  - Uses a small inline script to:
    - Sort `window.BLOG_POSTS` by `publishedOn` (newest first).
    - Render each post’s title, metadata, and `indexHtml` snippet.
    - Preserve existing Tailwind styling and layout.

- **Updated archive pages**:
  - `blog/archive/chatgpt-may-be-coming-to-carplay.html`
  - `blog/archive/meta-blames-apple-for-failure.html`
  - `blog/archive/apple-launches-creator-studio.html`
  - `blog/archive/tiktok-was-bought.html`
  - Each page:
    - Replaces hardcoded article markup with `<header id="post-header">`.
    - Loads `../posts.js`.
    - Uses a per-page script that:
      - Selects the correct entry from `window.BLOG_POSTS` by `slug`.
      - Injects the title, published date/author line, and `articleHtml` into the header.

### How to add or edit posts

1. **Add a new post**:
   - Open `blog/posts.js`.
   - Append a new object to `window.BLOG_POSTS` with the same shape as existing entries.
   - If the post has a dedicated archive page:
     - Set `hasArchive: true` and provide an `archivePath`.
     - Create a corresponding HTML file under `blog/archive/` that:
       - Includes `<header id="post-header">`.
       - Loads `../posts.js`.
       - Uses a script matching on the new `slug`.

2. **Edit an existing post**:
   - Update the `indexHtml` and/or `articleHtml` fields in `blog/posts.js`.
   - No changes are needed in `blog/index.html` or the archive pages as long as the `slug` and paths stay the same.
